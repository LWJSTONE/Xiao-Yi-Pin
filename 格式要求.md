> - 摘要内容必须包括以下四部分：
>   1. **目的**：为了解决什么问题，设计了一个什么系统

>   3. **功能**：系统包含哪些角色，主要有哪些功能（这些功能必须在论文后面的系统设计和实现中找得到）

> - 关键词数量：**3-5个**
> - 英文关键词使用**新罗马字体（Times New Roman）**

## 三、图片插入规范

### 3.1 图片位置与大小

- 图片应居中放置在正文中
- 图片宽度一般占页面版心宽度的 **70%~85%**（版心宽度约17cm，图片宽度约12cm~16cm）
- 图片高度根据内容自适应，但不宜过大
- 图片应在正文中首次提及后紧接着放置，或在该段落后紧跟

### 3.2 图片标题格式

- 图片标题位于图片**下方**
- 格式：**图X 图片描述**（如"图1 高校考试系统的用例图"、"图5 管理员登录界面"）
- 编号采用**阿拉伯数字连续编号**，从1开始，全文统一编号
- 字体：宋体，**五号（10.5pt）**，**加粗**
- 对齐方式：**居中对齐**
- 行距：1.5倍行距（18pt固定值）
- 图片标题与图片之间无空行或仅有小间距

### 3.3 图内字体要求

> **批注要求**：图中字体使用**宋体五号（10.5pt）**。

- 所有图片（包括用例图、架构图、流程图、时序图、E-R图、界面截图等）中出现的文字均应使用宋体五号字
- 确保图中文字清晰可读

### 3.4 正文引用格式

> **批注要求**：图出现之前应该有简单介绍，并有"如图X所示"。

- 在正文中引用图片时，使用"如图X所示"的格式
- **图片之前必须有文字介绍**，引用位置应在图片之前
- 示例："结合上述功能需求，得到了高校系统的用例图，如图1所示。"

### 3.5 图片类型

样张中包含的图片类型：
- 用例图
- 系统架构图
- 功能模块图（树状结构）
- E-R图（实体带属性，实体之间有联系）
- 界面截图
- 时序图（或流程图）

## 四、表格插入规范

### 4.1 表格标题格式

- 表格标题位于表格**上方**
- 格式：**表X 表格描述**（如"表1 题目表"、"表12 学生登录功能测试用例及结果"）
- 编号采用**阿拉伯数字连续编号**，从1开始，全文统一编号
- 字体：宋体，**五号（10.5pt）**，**加粗**
- 对齐方式：**居中对齐**

### 4.2 表格内容格式

- 表格内中文文字字体：宋体，小四号（12pt）
- 表格内英文文字字体：Times New Roman，小四号（12pt）
- 表格内文字对齐方式：**居中对齐**
- 表格边框：**上下双线边框**（三线表样式）
- 表格各列宽度根据内容自适应调整

### 4.3 正文引用格式

> **批注要求**：表出现之前要对其功能有介绍，并有"详见表X"。

- 在正文中引用表格时，使用"详见表X"的格式
- **表格之前必须有文字介绍**其功能或用途
- 示例："（1）题目表，主要用来保存题目的基本信息（详见表1）。"

### 4.4 数据库表设计规范

数据库表使用统一的列结构：
| 列名 | 说明 |
|------|------|
| 字段名 | 数据库字段名称 |
| 类型 | 数据类型（如int、longtext等） |
| 长度 | 字段长度 |
| 主键 | 是否为主键（是/否） |
| 备注 | 字段含义说明 |

每个数据库表上方应有文字说明，如：
- "（1）题目表，主要用来保存题目的基本信息（详见表1）。"

### 4.5 测试用例表格式

测试用例表使用统一的列结构：
| 列名 | 说明 |
|------|------|
| 序号 | 测试用例序号 |
| 测试描述 | 测试内容描述 |
| 输入 | 测试输入数据 |
| 预期结果 | 期望的测试结果 |
| 结果 | 实际测试结果（如"相符"） |

## 五、各章节内容要求

> 以下内容要求均来自样张**批注**。

### 5.1 绪论（第1章）

- 内容应包括：**研究背景**、**系统开发的目的和意义**
- 使用宋体小四字，英文用新罗马字体

### 5.2 基本技术介绍（第2章）

- 内容应包括：
  - 系统开发过程中所使用的**关键技术**
  - 系统的**开发工具和运行环境**
- **以上内容必须简明扼要，不能占太多篇幅**
- 使用宋体小四字，英文用新罗马字体

### 5.3 系统需求分析（第3章）

- 内容应包括：
  - **可行性分析**（技术可行性、操作可行性、经济可行性等）
  - **功能性需求分析**
  - **非功能性需求分析**
  - 必须画出**系统用例图**
- **以上内容描述务必详细、准确、完整**
- **用例图和功能需求描述要前后一致**
- 使用宋体小四字

### 5.4 系统总体设计（第4章）

- 内容应包括：
  - **系统架构设计**
    - 画出系统总体架构图
    - 云计算方向的系统架构图要有**注册中心**
  - **系统功能模块设计**
    - 画出**树状**的系统功能模块划分图
  - **数据库设计**
    - 数据库概念设计：画出正确的**E-R图**（**实体要带属性，实体之间有联系**）
    - 数据库逻辑结构设计
    - 数据库物理结构设计：表结构要和E-R图中实体**前后照应，不能凭空产生**
- 使用宋体小四字

### 5.5 系统详细设计与实现（第5章）

- 内容应包括：按顺序把主要功能模块**逐一介绍实现**
- **每个模块应包含以下四个部分**：
  1. **系统实现效果图**及相应文字解释
  2. **功能设计的时序图（或者流程图）**及相应文字解释
  3. **功能实现原理介绍**
  4. **关键实现代码**及必要解释
- 使用宋体小四，英文要用新罗马字体

### 5.6 系统测试（第6章）

- 内容应包括：
  - **功能性测试**
  - **非功能性测试**（如安全测试、性能测试等）
- 使用宋体小四，英文要用新罗马字体

#### 章节结构参考

```
6 系统测试
  ├── 6.1 系统测试的目的
  ├── 6.2 功能测试
  └── 6.3 非功能性测试（安全测试等）
       ├── 6.3.1 系统登录测试
       └── 6.3.2 系统权限测试
```

#### 测试要求

1. 每个测试模块需配有对应的测试用例表格
2. 测试用例表格格式统一（序号、测试描述、输入、预期结果、结果）
3. 正文描述与表格之间应紧密配合，文字说明在前，表格在后

### 5.7 总结（第7章）

- 总结自己毕业设计工作的**收获和感想**
- 以及对未来的**展望**
- **千万注意：总结部分不能再引用参考文献**（这是对自己毕业设计工作的总结和反思）

### 5.8 致谢（第8章）

- 致谢内容自由撰写

## 六、参考文献要求

### 6.1 标题格式

- "参考文献"作为独立章节标题（一级标题）
- 字体：黑体，20磅（20pt），**加粗**，居中
- 行距：单倍行距

### 6.2 参考文献条目格式

- 字体：宋体，小四号（12pt），英文用新罗马字体
- 每条参考文献独占一行
- 参考文献列表采用**悬挂缩进**格式（List Paragraph样式）

### 6.3 参考文献数量与质量要求

> **批注要求**：
> - 参考文献数量要**不低于10篇**
> - 至少**1篇英文文献**
> - 参考文献要**新，不能太陈旧**（近五年内为佳）
> - 参考文献**不能乱引用，一定要真引用**
> - **尤其在论文最后总结部分，千万不能再引用参考文献**

### 6.4 参考文献类型标识

| 类型标识 | 文献类型 |
|----------|----------|
| [M] | 专著/著作 |
| [J] | 期刊文章 |
| [D] | 学位论文 |
| [C] | 会议论文集 |
| [R] | 科技报告 |
| [N] | 报纸文章 |
| [S] | 标准 |
| [P] | 专利 |
| [EB/OL] | 电子文献 |

### 6.5 参考文献著录格式

**（1）著作 [M]**：
```
[序号] 作者（或主编）. 书名[M]. 出版地：出版单位，出版年份. 起止页码.
```
示例：
```
胡梦飞. 远载千秋：中国大运河传[M]. 北京：新华出版社，2025. 28-35.
```

**（2）期刊文章 [J]**：
```
[序号] 作者. 文章题目[J]. 刊名，年，卷（期）：起止页码.
```
示例：
```
卿朝进，凌国伟，王莉，等. 差分编码与神经网络辅助的OFDM系统信道估计方法[J]. 电子学报，2024, 52(06): 1852-1861.
```

**（3）学位论文 [D]**：
```
[序号] 作者. 论文名[D]. 学校所在城市：学校名，年份.
```
示例：
```
种展鹏. 氢气放电离子源中等离子体及其引出离子特性的模拟研究[D]. 西安：西安工业大学，2023.
```

**（4）英文文献**：
```
Author1, Author2, Author3, et al. Title[J]. Journal Name, Year, Volume(Issue): Pages.
```
示例：
```
Islam M M, Nabil S K, Murad S A, et al. The Development and Deployment of an Online Exam System: A Web Application[J]. Asian Journal of Research in Computer Science, 2023, 16(2): 1–11.
```

> **注意**：引用论文集[C]、学位论文[D]、报告[R]时格式类似，但文献类型标识不同。

### 6.6 参考文献排版注意事项

- 参考文献应包含**中文文献和英文文献**
- 中文文献在前，英文文献在后（或按正文引用顺序排列）
- 多作者文献使用"，"分隔，超过3人使用"等"（中文）或"et al."（英文）
- 文献排列按在正文中出现的先后顺序编号
- 参考文献的引用使用上标编号，如：[^1]

## 七、论文整体结构

根据样张及其批注，论文正文应包含以下章节：

| 章节 | 内容要点 |
|------|----------|
| 封面 | 文头、论文题目（禁止副题名）、作者信息（下划线） |
| 学生诚信承诺书 | 含作者、企业导师、校内导师、院长签名及日期 |
| 论文使用授权说明 | 含作者、企业导师、校内导师签名及日期 |
| 中文摘要 | 摘要（100-300字，含目的+技术+功能+优点）+ 关键词（3-5个） |
| 第1章 绪论 | 研究背景、系统开发的目的和意义 |

## 八、其他排版要求

### 8.1 行距

- 论文正文统一使用 **1.5倍行距**（固定值18pt）
- 部分标题可根据样式设置自动调整行距

### 8.2 首行缩进

- 正文段落首行缩进 **2字符**（约0.85cm）
- 标题不缩进
- 列表项可使用2字符或1.5字符缩进

### 8.3 段落间距

- 论文标题与作者之间段前段后间距约 **0.28cm**
- 章节标题与正文之间无额外空行

### 8.4 代码展示

- 代码部分使用等宽字体展示
- 代码段落不首行缩进
- 代码前应有文字说明（如"部分实现代码如下所示："）
- 代码后应有简要解释说明

### 8.5 编号格式

- 章节编号使用阿拉伯数字（1、2、3...）
- 子节编号格式：X.Y（如1.1、2.3）
- 三级标题编号格式：X.Y.Z（如2.1.1、4.3.2）
- 正文中的列举编号使用"（1）（2）（3）"格式

### 8.6 中英文字体使用原则

- 中文内容统一使用**宋体**（标题除外）
- 英文内容和数字统一使用 **Times New Roman（新罗马字体）**
- 章节标题使用**黑体**（中文部分）
