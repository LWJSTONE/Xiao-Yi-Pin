## 三、图片插入规范

### 3.1 图片位置与大小

- 图片应居中放置在正文中
- 图片宽度一般占页面版心宽度的 **70%~85%**（版心宽度约17cm，图片宽度约12cm~16cm）
- 图片高度根据内容自适应，但不宜过大
- 图片应在正文中首次提及后紧接着放置，或在该段落后紧跟

### 3.2 图片标题格式

- 图片标题位于图片**下方**
- 格式：**图X 图片描述**（如"图1 高校考试系统的用例图"、"图5 管理员登录界面"）
- 编号采用**阿拉伯数字连续编号**，按章节或全文顺序编号
- 字体：宋体，小四号（12pt），**加粗**
- 对齐方式：**居中对齐**
- 行距：1.5倍行距（18pt固定值）
- 图片标题与图片之间无空行或仅有小间距

### 3.3 正文引用格式

- 在正文中引用图片时，使用"如图X所示"的格式
- 引用位置应在图片之前，即先在文字中提及，再放置图片
- 示例："高校考试系统的用例图，如图1所示。"

### 3.4 图片类型

样张中包含的图片类型：
- 用例图
- 系统架构图
- 功能模块图
- E-R图
- 界面截图
- 时序图

## 四、表格插入规范

### 4.1 表格标题格式

- 表格标题位于表格**上方**
- 格式：**表X 表格描述**（如"表1 题目表"、"表12 学生登录功能测试用例及结果"）
- 编号采用**阿拉伯数字连续编号**
- 字体：宋体，小四号（12pt），**加粗**
- 对齐方式：**居中对齐**

### 4.2 表格内容格式

- 表格内文字字体：宋体，小四号（12pt）
- 表格内文字对齐方式：**居中对齐**
- 表格边框：**上下双线边框**（三线表样式）
- 表格各列宽度根据内容自适应调整

### 4.3 数据库表设计规范

数据库表使用统一的列结构：
| 列名 | 说明 |
|------|------|
| 字段名 | 数据库字段名称 |
| 类型 | 数据类型（如int、longtext等） |
| 长度 | 字段长度 |
| 主键 | 是否为主键（是/否） |
| 备注 | 字段含义说明 |

每个数据库表上方应有文字说明，如：
- "（1）题目表，主要用来保存题目的基本信息（详见表1）。"

### 4.4 测试用例表格式

测试用例表使用统一的列结构：
| 列名 | 说明 |
|------|------|
| 序号 | 测试用例序号 |
| 测试描述 | 测试内容描述 |
| 输入 | 测试输入数据 |
| 预期结果 | 期望的测试结果 |
| 结果 | 实际测试结果（如"相符"） |

## 五、系统测试部分要求

### 5.1 章节结构

系统测试作为独立的一章（第6章），包含以下小节：

```
6 系统测试
  ├── 6.1 系统测试的目的
  ├── 6.2 功能测试
  └── 6.3 安全测试
       ├── 6.3.1 系统登录测试
       └── 6.3.2 系统权限测试
```

### 5.2 系统测试的目的

- 说明系统测试在整个软件工程中的重要性
- 阐述系统测试的目的是验证最终软件系统是否满足用户规定的需求
- 说明通过系统测试可以及早发现并报告软件缺陷，提高软件的可靠性和稳定性

### 5.3 功能测试

- 针对系统的各个功能模块进行测试
- 每个功能模块的测试结果以**测试用例表格**形式呈现
- 测试用例表应包含：序号、测试描述、输入、预期结果、结果
- 文字描述应说明测试的背景、方法和预期
- 示例测试模块：
  - 学生登录功能测试
  - 教师管理模块测试（新增教师等）
  - 题库管理模块测试

### 5.4 安全测试

- 说明安全测试的核心目标：发现潜在的安全隐患
- 分为以下子测试：
  - **系统登录测试**：验证登录时数据比对和错误提示功能
  - **系统权限测试**：验证不同角色的权限控制和页面访问控制
- 每项安全测试同样以**测试用例表格**形式呈现

### 5.5 测试要求总结

1. 每个测试模块需配有对应的测试用例表格
2. 测试用例表格格式统一（序号、测试描述、输入、预期结果、结果）
3. 正文描述与表格之间应紧密配合，文字说明在前，表格在后
4. 测试结果应体现系统的功能正确性和安全性

## 六、参考文献要求

### 6.1 标题格式

- "参考文献"作为独立章节标题（一级标题）
- 字体：黑体，三号（16pt），**加粗**，居中
- 行距：单倍行距

### 6.2 参考文献条目格式

- 字体：宋体，小四号（12pt）
- 每条参考文献独占一行
- 参考文献列表采用**悬挂缩进**格式（List Paragraph样式）
- 参考文献数量：样张中为**10篇**，建议不少于10篇

### 6.3 参考文献类型标识

| 类型标识 | 文献类型 |
|----------|----------|
| [J] | 期刊文章 |
| [D] | 学位论文 |
| [C] | 会议论文集 |
| [N] | 报纸文章 |
| [M] | 专著 |
| [R] | 科技报告 |
| [S] | 标准 |
| [P] | 专利 |
| [EB/OL] | 电子文献 |

### 6.4 参考文献著录格式

**期刊文章 [J]**：
```
作者. 文章题目[J]. 期刊名, 年份, 卷(期): 起止页码.
```
示例：
```
康昊. 微服务架构技术与挑战[J]. 网络安全技术与应用, 2023(02): 3-4.
```

**学位论文 [D]**：
```
作者. 论文题目[D]. 城市：学校名, 年份.
```
示例：
```
涂腾飞.Go语言中的并发问题研究[D].北京：北京邮电大学,2019.
```

**英文文献**：
```
Author1, Author2, Author3, et al. Title[J]. Journal Name, Year, Volume(Issue): Pages.
```
示例：
```
Islam M M, Nabil S K, Murad S A, et al. The Development and Deployment of an Online Exam System: A Web Application[J]. Asian Journal of Research in Computer Science, 2023, 16(2): 1–11.
```

### 6.5 参考文献注意事项

- 参考文献应包含**中文文献和英文文献**
- 中文文献在前，英文文献在后（或按正文引用顺序排列）
- 多作者文献使用"，"分隔，超过3人使用"等"（中文）或"et al."（英文）
- 文献排列按在正文中出现的先后顺序编号

## 七、论文整体结构

根据样张，论文正文应包含以下章节：

| 章节 | 内容 |
|------|------|
| 摘要 | 中文摘要 + 关键词 |
| 第1章 绪论 | 研究背景、系统开发的目的和意义等 |
| 第2章 基本技术介绍 | 开发技术简介、开发工具和运行环境 |
| 第3章 系统需求分析 | 可行性分析、功能性需求分析、非功能性需求分析 |
| 第4章 系统总体设计 | 系统总体架构设计、系统功能模块划分、数据库设计 |
| 第5章 系统详细设计与实现 | 各模块的详细设计与实现（含界面截图、时序图、核心代码） |
| 第6章 系统测试 | 系统测试的目的、功能测试、安全测试 |
| 第7章 总结 | 总结与展望 |
| 第8章 致谢 | 致谢内容 |
| 参考文献 | 参考文献列表 |
| 英文摘要 | 英文Abstract + Key words |

## 八、其他排版要求

### 8.1 行距

- 论文正文统一使用 **1.5倍行距**（固定值18pt）
- 部分标题可根据样式设置自动调整行距

### 8.2 首行缩进

- 正文段落首行缩进 **2字符**（约0.85cm）
- 标题不缩进
- 列表项可使用2字符或1.5字符缩进

### 8.3 段落间距

- 论文标题与作者之间段前段后间距约 **0.28cm**
- 章节标题与正文之间无额外空行

### 8.4 代码展示

- 代码部分使用等宽字体展示
- 代码段落不首行缩进
- 代码前应有文字说明（如"部分实现代码如下所示："）
- 代码后应有简要解释说明

### 8.5 编号格式

- 章节编号使用阿拉伯数字（1、2、3...）
- 子节编号格式：X.Y（如1.1、2.3）
- 三级标题编号格式：X.Y.Z（如2.1.1、4.3.2）
- 正文中的列举编号使用"（1）（2）（3）"格式
